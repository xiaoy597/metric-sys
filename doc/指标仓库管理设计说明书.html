  • 指标维度字典表

    指标维度字典表记录各指标维度的各级粒度的维度代码及代码的中文说明。维度字典表的命名方式为：<维度物理表名称>_<粒度物理字段名称>。例如，地区维度（admin_dim）中的城市粒度（d_city）所对应的指标维度字典表为：admin_dim_d_city。由于指标维度和粒度是根据用户需要而动态变化的，维度字典表需要根据维度和粒度的变化而动态创建。

    指标维度表、指标主表和指标维度字典表都存在于指标仓库中。

    维度字典表的结构为：

    字段名称 字段中文名称 字段描述 主键

    code_value

    维度粒度值代码

    是

    code_desc

    维度粒度值描述

    维度粒度值的中文描述，数据类型统一为VARCHAR(80）

    否

功能列表

指标模型浏览

用户界面交互

  1. 页面左侧显示指标仓库对象树。对象分为两类：

    1. 指标维度。按照指标维度代码的次序显示各维度的中文名称。

    2. 指标主表。按照指标主表代码的次序显示各主表的中文名称及该主表所关联的维度名称（维度按照维度代码排序）。

  2. 用户点击指标仓库对象树中的维度名称节点，右侧显示该维度的属性页面。

    1. 维度粒度标签。显示该维度的粒度列表，按粒度级别排序。

    2. 维度数据标签。根据该维度的表结构显示维度物理表的数据。

  3. 用户点击指标仓库对象树中的主表名称节点，右侧显示该主表的属性页面。

    1. 主表字段列表。显示主表所有度量字段的属性，按度量字段代码排序。

后台处理逻辑

  1. 查询指标维度列表

  2. 查询指标主表列表

  3. 查询指标维度属性

  4. 查询指标维度数据

  5. 查询指标主表属性

新增指标维度

用户界面交互

  1. 填写下列属性信息：

    1. 维度代码：4位数字，不得与现有代码重复。

    2. 维度名称：不得与现有维度名称重复。

    3. 维度物理表名称：不得是指标仓库中已存在的表。

    4. 维度粒度列表：

      1. 粒度名称：不得与该维度其他粒度的名称重复。

      2. 粒度字段代码：D+9位数字，不得与现有指标字段代码重复。

      3. 粒度物理名称：合法的数据库字段名，不得与该维度其他粒度的字段物理名称重复。

      4. 粒度描述信息：对粒度业务含义的说明。

      5. 新增粒度的粒度级别默认为当前最大级别+1。

后台处理逻辑

  1. 保存维度及维度粒度数据，并对各属性的合法性（判断重复）进行核对检查。

维度粒度级别调整

用户界面交互

  1. 打开维度属性页面的维度粒度标签页

  2. 在维度粒度列表中，点击“上移”使该粒度的级别减1，直到级别变为0；点击“下移”使该粒度级别加1，直到成为级别最大的粒度。

  3. 保存用户修改前，判断维度被已生效指标引用的次数，如果大于0次，则提示“维度正在被使用，无法修改”。

后台处理逻辑

  1. 将用户调整后的维度粒度级别保存到维度粒度表（metric_dim_col）中。

  2. 检查维度被已生效指标引用的次数。

修改指标维度

用户界面交互

  1. 打开现有维度的属性页面

  2. 对维度和维度粒度的属性进行修改，规则与新增指标维度相同。

  3. 保存用户修改前，判断维度被已生效指标引用的次数，如果大于0次，则提示“维度正在被使用，无法修改”。

后台处理逻辑

  1. 将修改后的维度属性保存到维度表（metric_dim）中。

  2. 将修改后的维度粒度属性保存到指标字段表（metric_column）和维度粒度表（metric_dim_col）中。

  3. 检查维度被已生效指标引用的次数。

维护指标维度数据

用户界面交互

  1. 打开现有维度的属性页面

  2. 点击维度数据标签页。

  3. 新增维度数据：各粒度字段不得有空值。

  4. 修改维度数据：需要判断该维度的指标引用次数，如果不为0，则提示“维度正在被使用，无法修改”。

  5. 删除维度数据：需要判断该维度的指标引用次数，如果不为0，则提示“维度正在被使用，无法修改”。

后台处理逻辑

  1. 将修改后的维度数据保存到对应的维度表中。

  2. 保存后，调用后台的指标维度重构接口对指标维度数据进行重构。

    curl -k https://<metric-engine-ip>:<metric-engine-port>/metric-engine/build/<dimension_cd>;

维护指标粒度字典

用户界面交互

  1. 打开现有维度的属性页面

  2. 点击维度粒度列表中的“字典”按钮。

  3. 对维度粒度字典表数据进行修改，修改内容不做限制

后台处理逻辑

  1. 将修改后的维度粒度字典保存到对应的维度粒度字典表中。

  2. 查询维度粒度字典数据。

功能列表

数据备份

用户界面交互

  1. 进入数据备份页面，显示数据备份列表

  2. 查询系统运行状态，如果不是维护状态，将“新备份”按钮禁用。

  3. 点击“新备份”，填写备份备注信息，点击“开始”。

  4. 刷新备份列表。

后台处理逻辑

  1. 查询系统运行状态：系统参数表（metric_param），参数名称为：SYSTEM_RUN_STATE，值为：NORMAL - 正常状态，MAINTENANCE - 系统维护。

  2. 查询数据备份列表。

  3. 将新备份的信息插入指标数据备份表（metric_data_backup），然后调用后台数据备份服务：

    curl -k -XPOST https://<metric-keeper-ip>:<metric-keeper-port>/metric-keeper/backup -d '{"op":"backup", "id": <backup_id>}'
